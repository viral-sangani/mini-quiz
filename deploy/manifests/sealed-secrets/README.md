# Sealed Secrets

The `sealed-secrets` controller (deployed via `deploy/apps/sealed-secrets.yaml`)
holds a cluster-private RSA key. We encrypt secrets locally with the matching
public key (`kubeseal`) and commit the encrypted YAML here. Only the cluster
can decrypt.

## Generate the api Secret

```bash
# 1. Pull the cluster's public cert (one-time per cluster)
kubeseal --fetch-cert \
  --controller-namespace kube-system \
  --controller-name sealed-secrets \
  > /tmp/sealed-secrets.pub

# 2. Build a plain Secret manifest locally — DO NOT COMMIT THIS
cat <<'EOF' > /tmp/api-secret.yaml
apiVersion: v1
kind: Secret
metadata:
  name: api-secrets
  namespace: api
type: Opaque
stringData:
  NEXTAUTH_SECRET: "..."
  ADMIN_EMAILS: "you@example.com"
  TREASURY_PRIVATE_KEY: "0x..."
  CELO_RPC_URL: "https://forno.celo.org"
  CORS_ORIGIN: "https://miniquiz.club,https://admin.miniquiz.club"
  LOG_LEVEL: "info"
EOF

# 3. Seal it
kubeseal --format yaml --cert /tmp/sealed-secrets.pub \
  < /tmp/api-secret.yaml \
  > deploy/manifests/sealed-secrets/api-secrets.yaml

# 4. Commit the sealed file. Delete the plaintext.
rm /tmp/api-secret.yaml
```

## Generate the redis-auth Secret

```bash
cat <<'EOF' > /tmp/redis-auth.yaml
apiVersion: v1
kind: Secret
metadata:
  name: redis-auth
  namespace: data
type: Opaque
stringData:
  password: "$(openssl rand -base64 24)"
EOF

kubeseal --format yaml --cert /tmp/sealed-secrets.pub \
  < /tmp/redis-auth.yaml \
  > deploy/manifests/sealed-secrets/redis-auth.yaml

rm /tmp/redis-auth.yaml
```

## DATABASE_URL composition

CNPG auto-generates `Secret/miniquiz-pg-app` in the `data` namespace with
`username` / `password` / `uri` keys. The api chart references that Secret
directly (see `deploy/charts/api/templates/deployment.yaml`'s `envFrom`),
so DATABASE_URL is **not** part of `api-secrets`.

## Files in this dir

- `api-secrets.yaml` — encrypted; safe to commit (generated by you locally).
- `redis-auth.yaml` — encrypted; safe to commit.
- `README.md` — this file.

Plaintext secrets must never land in git.
